搜索算法Ⅲ-Local Search

Searching Problem的分类与定义

搜索的问题根据搜索的条件、限制和目标的不同，可以有以下几类定义:
1. Uninformed Search(盲目搜索)
2. Informed Search(知情搜索)
3. Local Search(局部搜索)

要解决搜索问题，首先需要明白如何去定义这个问题
1. States: 构成状态的详细信息
2. Initial state: 初始状态
3. Actions: 可以进行的操作
4. Transition: 状态的转移
5. Cost: 路径的成本函数
解决搜索问题的方案就是将初始状态转换为目标状态的一系列动作。


Local Search

Local Search局部搜索从一个初始解出发，然后搜索解的邻域，如有更优的解则移动至该解并继续执行搜索，否则返回当前解。局部搜索适合用于寻找目标状态的问题，具有所需内存低，能在大的空间中找到解决方案的优点。

Local Search通常伴随着搜索的限制满足条件(Constraint Satisfaction Problems)，在逐步构建解决方案的过程中，如果其中一种解决方案不满足限制条件，则立即放弃该解，进行回溯。


Backtracking Search


Backtracking Search回溯是一种递归解决问题的算法技术，通过尝试逐步构建解决方案，删除那些在任何时间点都不能满足问题约束的解决方案。

以组合总数这个问题为例，一个无重复元素的整数数组 candidates 和一个目标整数 target ，找出 candidates 中可以使数字和为目标数 target 的所有不同组合 ，并以列表形式返回。

candidates 中的同一个数字可以无限制重复被选取 。如果至少一个数字的被选数量不同，则两种组合是不同的。 

// combination-sum
vector<vector<int>> ans;
vector<int> path;
void backTrack(vector<int>& candidates, int idx, int target, int sum) {
   if(sum == target) {
      ans.push_back(path);
      return;
   }
   else if(sum > target) {
      return;
   }
   // 从idx开始
   for(int i = idx; i < candidates.size(); i++) {
      path.push_back(candidates[i]);
      sum += candidates[i];
      backTrack(candidates, i, target, sum); // i表示该数字可以无限制被重复选取
      path.pop_back();
      sum -= candidates[i];
   }
}
vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
   backTrack(candidates, 0, target, 0);
   return ans;
}

Backtracking Search+Pruning

Pruning剪枝就是在搜索过程中来剪去不符合限制条件的搜索路径，来避免了一些不必要的搜索。

以组合总数Ⅱ这个问题为例，给定一个候选人编号的集合 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。

candidates 中的每个数字在每个组合中只能使用一次，且解集不能包含重复的组合。

// combination-sum-ii
vector<vector<int>> ans;
vector<int> path;
vector<bool> visited;
void backTrack(vector<int>& candidates, int idx, int target, int sum) {
   if(sum == target) {
      ans.push_back(path);
      return;
   }
   else if(sum > target) {
      return;
   }
   for(int i = idx; i < candidates.size(); i++) {
      // 如果该元素和前一个元素相等，且前一个元素没有使用，进行剪枝，避免重复
      if(i > 0 && candidates[i] == candidates[i - 1] && visited[i - 1] == false) {
         continue;
      }
      path.push_back(candidates[i]);
      sum += candidates[i];
      visited[i] = true;
      backTrack(candidates, i + 1, target, sum);
      path.pop_back();
      sum -= candidates[i];
      visited[i] = false;
   }
}
vector<vector<int>> combinationSum2(vector<int>& candidates, int target) {
   // 对数组进行排序
   sort(candidates.begin(), candidates.end());
   // 记录数组中元素的是否被使用过
   visited = vector(candidates.size(), false);
   backTrack(candidates, 0, target, 0);
   return ans;
}

N-Queens

接下来看一下N皇后这个经典的回溯问题。


N皇后问题研究的是如何将N个皇后放置在 n×n 的棋盘上，并且使皇后彼此之间不能相互攻击。

给定一个整数 n ，返回所有不同的 n 皇后问题的解决方案。

// N-Queues
vector<int> queens;
unordered_set<int> columns;
unordered_set<int> diagonals;
unordered_set<int> off_diagonals;
vector<vector<string>> ans;

void backTrack(int row, int n) {
   if(row == n) {
      generateAns(queens, n);
      return;
   }
   for(int i = 0; i < n; i++) {
      if(columns.find(i) != columns.end()) {
         continue;
      }
      int d = row - i;
      if(diagonals.find(d) != diagonals.end()) {
         continue;
      }
      int off_d = row + i;
      if(off_diagonals.find(off_d) != off_diagonals.end()) {
         continue;
      }
      queens[row] = i;
      columns.insert(i);
      diagonals.insert(d);
      off_diagonals.insert(off_d);
      backTrack(row + 1, n);
      queens[row] = -1;
      columns.erase(i);
      diagonals.erase(d);
      off_diagonals.erase(off_d);
   }
}

vector<vector<string>> solveNQueens(int n) {
   queens = vector<int>(n);
   backTrack(0, n);
   return ans;
}

void generateAns(vector<int> &queens, int n) {
   vector<string> v(n);
   for(int i = 0; i < n; i++) {
      string str(n, '.');
      int row = queens[i];
      str[row] = 'Q';
      v[i] = str;
   }
   ans.push_back(v);
}
