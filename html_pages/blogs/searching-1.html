搜索算法Ⅰ-Uninformed Search

Searching Problem的分类与定义

搜索的问题根据搜索的条件、限制和目标的不同，可以有以下几类定义:
1. Uninformed Search(盲目搜索)
2. Informed Search(知情搜索)
3. Local Search(局部搜索)

要解决搜索问题，首先需要明白如何去定义这个问题
1. States: 构成状态的详细信息
2. Initial state: 初始状态
3. Actions: 可以进行的操作
4. Transition: 状态的转移
5. Cost: 路径的成本函数
解决搜索问题的方案就是将初始状态转换为目标状态的一系列动作。


Uninformed Search

Uninformed Search(盲目搜索)是指除了问题本身的定义之外，没有其他附加信息的一种搜索。


Breadth-first search(BFS)


广度优先搜索BFS的搜索策略是先探索在搜索空间中比较“浅”的节点。

考虑下面的状态空间图，S是搜索的起始节点，G是搜索的目标节点，假设一个节点的子节点的入队顺序是按照字母来排的。

在实现时，可以使用set来记录已经探索过的节点，避免重复计算。


上图中，S为起始节点，G为目标节点，连线表示两点能够连通，需要解决的问题是找到从S到G的路径。

// BFS
import collections
def bfsGsa(stateSpaceGraph, startState, goalState): 
      frontier = collections.deque([startState])
      exploredSet = set()
      print('Initial frontier:',list(frontier))
      input()
      while frontier: 
         node = frontier.popleft()
         if (node.endswith(goalState)): return node
         if node[-1] not in exploredSet:
            print('Exploring:',node[-1],'...')
            exploredSet.add(node[-1])
            for child in stateSpaceGraph[node[-1]]: frontier.append(node+child)
            print(list(frontier))
            print(exploredSet)
            input()
practice = {
      'S':['a','d','G'],'a':['b'], 'd':['b','e'],'G':[],'b':['e','c'],
      'e':['G'],'c':['G']}
print('Solution path:',bfsGsa(practice, 'S', 'G'))

Depth-first search(DFS)


深度优先搜索DFS的搜索策略是先探索在搜索空间中比较“深”的节点。

// DFS
import collections
def dfsGsa(stateSpaceGraph, startState, goalState): 
      frontier = collections.deque([startState])
      exploredSet = set()
      print('Initial frontier:',list(frontier))

         node = frontier.pop()

            print(list(frontier))

Uniform-cost search(UCS)


UCS指的搜索策略是先探索在搜索空间中cost最低的节点，在代码实现时，需要使用到优先队列这个数据结构，cost低的点优先出栈。

// UCS
from heapq import heappush, heappop
def ucsGsa(stateSpaceGraph, startState, goalState): 
      frontier = []
      heappush(frontier, (0, startState))
      exploredSet = set()
      print('Initial frontier:',list(frontier)); input()
      while frontier:
         node = heappop(frontier)
         if (node[1].endswith(goalState)): return node
         if node[1][-1] not in exploredSet:
            print('Exploring:',node[1][-1],'at cost',node[0])
            exploredSet.add(node[1][-1])
            for child in stateSpaceGraph[node[1][-1]]:
                  heappush(frontier, (node[0]+child[0], node[1]+child[1]))
            print(list(frontier)); print(exploredSet); input()
practice = {
      'S':[(3,'a'),(2,'d'),(10,'G')],'a':[(5,'b')],
      'd':[(1,'b'),(4,'e')],'G':[],'b':[(1,'e'),(2,'c')],
      'e':[(3,'G')],'c':[(4,'G')]}
print('Solution path:',ucsGsa(practice, 'S', 'G'))

Uninformed Search方法对比

当目标节点离根节点近时，BFS优于DFS。

当目标节点很深时，DFS优于BFS。

当可以得到代价函数时，UCS能更快地以更小的代价找到目标节点。
